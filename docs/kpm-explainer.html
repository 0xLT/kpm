What A Knowledge Package Is

A package is not code. It is a versioned graph of Markdown files with explicit metadata, public exports, dependency links, and optional compiled context outputs.

Manifest

Every package has `knowledge.json`

The manifest defines the package name, version, Markdown entrypoints, public exports, wikilink policy, and dependencies.

Graph

Markdown files link with `[[wikilinks]]`

Links can point to nearby notes, headings, aliases, or exported files from installed dependency packages.

Compiler

Builds produce curated context

`kpm build` starts from entry files, follows links, removes duplicates, respects token budgets, and writes clean outputs.

How It Works

The MVP uses public GitHub repositories as the registry. A GitHub repo becomes installable when it contains a valid `knowledge.json` and linked Markdown files.

1

Initialize

Create a local knowledge project with a manifest, README, and index note.

2

Install

Add a package from `github:owner/repo#ref` or a local `file:` source.

3

Resolve

Parse wikilinks, enforce exports, and validate headings across packages.

4

Diagnose

Run `kpm doctor` to catch missing notes, ambiguous links, and invalid manifests.

5

Build

Compile selected notes into context files for AI coding tools and agents.

Package Format

The manifest is the public contract. It keeps packages composable by exposing only the notes that other projects are allowed to reference. 

{
  "name": "@acme/react-guide",
  "version": "0.1.0",
  "type": "knowledge-package",
  "exports": {
    ".": "./README.md",
    "./components": "./components.md",
    "./queries": "./queries.md"
  },
  "context": {
    "entrypoints": ["README.md"],
    "include": ["**/*.md"],
    "exclude": ["drafts/**", "private/**"],
    "requireWikilinks": true
  },
  "knowledgeDependencies": {
    "@acme/sql-guide": "github:owner/sql-guide-context#v0.2.0"
  }
}

Exports are the public API

`[[@acme/react-guide/components]]` works because `./components` is exported. Private files stay private unless the author exposes them.

Wikilinks are required by default

`kpm doctor` expects Markdown files to participate in the graph. Leaf notes can opt out with `kpmAllowNoWikilinks: true` frontmatter.

How To Use It

The current MVP is a local CLI. Use GitHub repos as the package transport, then compile project context for the AI tool you want to feed. 

# Install dependencies and build the CLI
npm install
npm run build

# Create a local knowledge package
./dist/cli.js init --name @company/project-context

# Install a public GitHub-backed package
./dist/cli.js add github:acme/react-context#v0.1.0

# Install a local package while authoring
./dist/cli.js add file:/absolute/path/to/context-pack

# Validate the graph before building
./dist/cli.js doctor
./dist/cli.js graph

# Compile context for different consumers
./dist/cli.js build --entry README.md --max-tokens 50000
./dist/cli.js build --target cursor
./dist/cli.js build --target claude
./dist/cli.js build --target rag

Build Outputs

`kpm build` turns a package graph into artifacts that can be pasted, uploaded, indexed, or checked into tool-specific locations.

Default outputs

dist/context.md The compiled Markdown context bundle.
dist/graph.json Resolved note and wikilink graph.
dist/citations.json Source records for every file included in the build.

Targeted outputs

--target cursor Writes `.cursor/rules/kpm-context.mdc`.
--target claude Writes `dist/claude-project-context.md`.
--target llms-txt Writes `dist/llms.txt`.
--target rag Writes JSONL chunks and an embeddings manifest.
!

Trust Model

Knowledge packages influence model behavior, so treat them like supply-chain inputs. Prefer pinned GitHub refs, inspect diffs before upgrading, run `kpm doctor`, and keep private or draft notes out of public exports.

kpm is a local-first MVP for versioned context packs. The next product step is validating real public GitHub packages end to end.
